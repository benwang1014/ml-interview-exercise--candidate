# ML Interview Exercise

## Overview
This repository contains a small dataset of questions and associated tags. The goal is to build an end-to-end machine learning model that can predict relevant tags for a new, unseen question.

This is intended to be a **collaborative discussion** where we explore your solution together. We are interested in how you think, reason, and make tradeoffs—not in perfection.

## Timeline
- You will receive this exercise **one day before** your interview
- Please complete the exercise **before** the interview
- During the 1-hour interview, you'll walk us through your solution and discuss your approach

## The Task
Build an end-to-end model that predicts one or more tags given a question. This includes:
- Data exploration and preprocessing
- Model training
- Evaluation
- Ability to make predictions on new questions

### Example
Given a new question like: *"What is the cost per semester?"*  
Your model should predict relevant tags such as: `cost`, `per-semester`, or similar.

### You May
- Use any libraries, frameworks, or tools (e.g., scikit-learn, transformers, pandas, ChatGPT, GitHub Copilot, Claude AI)
- Use any modeling approach you think is appropriate
- Modify or add files as you see fit
- Use any language you are comfortable with
- Leave TODOs or notes about what you would do with more time

## Dataset
The dataset is located in `data/question-tags-sample-data.csv`

### Columns
- `question` – text of the question
- `tags` – associated tags (may be one or multiple)

You are free to inspect the data and decide how to model the problem.

## Submission
Please share your solution with us so we can review it later. Fork this repo and share the link during the solution walkthrough.

## What We Value
During the solution walkthrough, we're interested in understanding:
- How you approached the problem
- Your decision-making process and tradeoffs
- How you evaluate and interpret results

We care more about your reasoning than perfect code or metrics.

We are **not** evaluating:
- Speed or time to completion
- Memorization of APIs or syntax
- State-of-the-art model performance
- Production-ready code

## During the Interview
Be prepared to:
- Run your solution and demonstrate predictions
- Walk through your code and explain your approach
- Discuss what worked, what didn't, and why
- Answer questions about alternative approaches
- Discuss what you would do differently with more time or resources

## Notes
- **Perfection is not expected**—we know you have limited time
- Focus on demonstrating your thought process rather than achieving the best possible metrics
- It's completely fine to acknowledge limitations or areas for improvement
